# Privacy Policy

**Last Updated:** 2026-07-09

**SkillX** is a local-first desktop application. Your skills, configurations, and files stay on your device. We do not operate any backend service that collects personal data.

## 1. Data We Do NOT Collect

We do **not** collect, transmit, or share:
- Personal information
- Usage analytics or behavior tracking
- Your skills content or file contents
- File system structure

## 2. Local Data Storage

All data generated by SkillX is stored on your local device:
- **App configuration**: `~/.skillx/config.json` (may contain LLM Provider API keys you added yourself)
- **Translation cache**: `~/.skillx/cache/translations/`
- **Skills content**: in the directories you configure

## 3. Network Requests

SkillX is designed to work primarily offline, but the following features **do** make outbound network requests. These calls are triggered by explicit user actions and do not send analytics:

| Feature | When | Destination |
|---------|------|-------------|
| **Update check** | Manual click or on startup (optional) | GitHub Releases (`api.github.com`) |
| **Marketplace browse / install** | You open Marketplace or install a skill | Configured marketplace source (default: public GitHub repos) |
| **AI translation / AI classification** | You click translate or classify | The LLM Provider **you configured** (OpenAI, DeepSeek, your own Ollama, etc.) |
| **Provider model listing** | You add or test a Provider | The Provider's base URL |
| **Sign-in (optional)** | You use OAuth to unlock account features | GitHub OAuth |
| **Feedback submission** | You submit feedback | GitHub Issues API |

**Nothing is sent unless you trigger it.** In particular, we do not run background telemetry.

## 4. Third-Party AI Providers

When you use the AI translation / classification features, your skill content and the LLM Provider's API Key you configured are sent **directly** to that Provider's endpoint (e.g. OpenAI). SkillX does not proxy or store this traffic. Please refer to your chosen Provider's own privacy policy.

## 5. Third-Party AI Tools SkillX Manages

SkillX creates symbolic links inside the config directories of tools like Claude Code, Codex, Gemini, Cursor, etc. When you run those tools, they read the linked skill files under their own privacy policies — SkillX has no visibility into that.

## 6. Your Data, Your Control

- The `~/.skillx/` directory is fully under your control — you can back it up, migrate it, or delete it at any time.
- Deleting the SkillX app does not delete `~/.skillx/` — remove it manually if desired.
- API keys you enter are stored in plaintext in `config.json`. **Do not share this file.**
