# 2026-06-26 · SkillX 4 套主题集成 (v2 加强)

## 集成机制
- 风格 5 选 1（默认/简约/科幻/轻拟/漫画）+ 明暗 3 选 1 = 15 种皮肤组合
- 动态 link 替换 + html `data-theme-style` + `.dark` 双轨制
- shadcn 变量桥接层（36+ 变量）

## v2 加强：风格特征穿透
**问题**：v1 集成后 4 套主题"差异不大"——SkillX 现组件大量用 Tailwind 硬编码类
（`rounded-xl border bg-white text-sm`），特异性高于主题 `[data-theme-style]`
选择器，导致主题视觉特征被压在下面。

**解决**：新增 `src/themes/_enhance.css`（8KB 加强层），用更高特异性的
"主题+元素"组合选择器，覆盖 7 大组件硬编码类：
- Sidebar nav（NavLink 渲染的 a）：胶囊 vs 圆角卡片 vs 粗墨线 vs L 角辉光 vs 凹陷
- Card（`bg-card rounded-xl` 等）：背景/边框/圆角/阴影全部接管
- Button（`bg-primary` / `bg-secondary` / `bg-destructive` / `border-input`）：4 个 variant 全覆盖
- Badge（`bg-primary rounded-md` 等）：3 个 variant
- Input（`border-input`）：含 placeholder
- Alert（`role="alert"`）：含 destructive 变体
- Switch（`role="switch"`）：data-state=checked
- 通用 Typography：h1/h2/h3 走 --font-display，body 走 --font-body
- 滚动条：thumb 颜色

**4 套风格专属强化**（每套独立 ~30 行 CSS）：
- cyberpunk：青色辉光 + L 角 `::before/::after` + 扫描线 + 雷达网格 + HUD 字体大写 + scale(0.95)
- comic：3px 粗黑边 + 5px 5px 0 0 硬偏移阴影 + hover translate(-2px,-2px) 阴影变色 + Bangers 字体
- apple：白卡 + 浅边 + 圆角胶囊按钮 + 1.5rem 大圆角 + hover 上浮
- neumorphism：暖亚麻 + 双向阴影 + 顶部 1px 高光 + inset 凹陷 + 衬线 display

## 关键文件
- `src/themes/_enhance.css`（新建 ~8KB）— 加强层
- `src/hooks/useTheme.tsx` — 加 1 个 useEffect 在 mount 时注入 `#theme-enhance` link

## 加载顺序（关键）
1. `_enhance.css`（高特异性兜底，先加载）
2. 主题文件（`{style}-{mode}/theme.css`，特异性中等）
3. 组件硬编码（`rounded-xl bg-white` 等，低特异性，最后渲染但被前 2 覆盖）

加强层在主题文件**之后**加载会出问题：它会盖住主题。已确保 enhance.css
由 `useTheme` 在 mount 时注入 `<link id="theme-enhance">`，**晚于主题
文件**加载，所以主题的 [data-theme-style] 仍能控制 `_enhance.css` 内的
覆盖（因为 enhance.css 内部用的就是 `var(--color-*)`）。

## v3 加强：主题特征深度穿透（macOS Vibrancy 默认风格 + 形态 token）

**问题**：v2 后 5 套主题仍"差异不大"。根本原因：shadcn cva 文件硬编码 `rounded-xl` `shadow-sm` `rounded-md` `border` 等类，specificity 高于主题 token。

**v3 决策**：不碰 7 个 cva 文件（避免 TypeScript 回归），而是**新增 4 个形态 token + 强化 `_enhance.css`**。

### 新增 token（在 `index.css :root` 兜底 + 10 主题 CSS 各覆盖）

| Token | 默认值 | 用途 |
|---|---|---|
| `--radius-card` | 0.75rem | Card 圆角（替代硬编码 `rounded-xl`）|
| `--radius-button` | 0.5rem | Button 圆角（替代 `rounded-md`）|
| `--border-w-card` | 1px | Card 边框宽度（替代裸 `border`）|
| `--shadow-toast` | `0 4px 12px rgba(0,0,0,0.15)` | Toast 阴影（替代内联硬编码）|

### 各主题形态 token 表

| 主题 | radius-card | radius-button | border-w-card | shadow-toast |
|------|-------------|---------------|---------------|--------------|
| default | 12px | 6px | 1px | 通用阴影 |
| apple（简约）| **24px** | **9999px** (pill) | 1px | 8px 24px 柔阴影 |
| cyberpunk（科幻）| **4px** (HUD) | **2px** (sharp) | 1px | **0 0 20px 青色辉光** |
| neumorphism（轻拟）| **28px** | **9999px** (pill) | 1px | 12px 32px 暖柔阴影 |
| comic（漫画）| **6px** | **4px** | **3px 粗墨线** | **5px 5px 0 硬偏移** |

### `_enhance.css` v3 追加（4 段共 ~100 行）

1. **Card/Button 圆角桥**：`[class*="bg-card"]` → `var(--radius-card) !important`
2. **Toast 主题化**：`[data-sonner-toast]` + `data-type="error/success/info"` → 6 段覆盖内联硬色
3. **Loading Skeleton 圆角桥**：4 段匹配 `style*="borderRadius: 16/14/6/4"`
4. **Layout 同步 banner 文字色修复**：`bg-yellow` 内 `text-white` button → `var(--color-banner-fg)`

### 默认风格升级为 macOS Vibrancy

按 STYLEKIT macOS-vibrancy 提示词改写 `default-light/dark`：
- 三层灰：`#ECECEE → #FFFFFF → #F2F2F7`（亮）/ `#1C1C1E → #2C2C2E → #3A3A3C`（暗）
- 1px 半透边：`rgba(0,0,0,0.08)` / `rgba(255,255,255,0.08)`
- 极克制阴影：`0 1px 2px rgba(0,0,0,0.04)`（亮）/ `0 1px 0 rgba(255,255,255,0.05)`（暗）
- macOS 语义色：`#007AFF` 蓝、`#FF3B30` 红、`#34C759` 绿、`#FF9500` 橙
- 衬线 display：`Charter / Iowan Old Style / Songti SC`
- 无渐变无发光，1px 哲学

## v3.1 修复：页面内联 borderRadius 穿透（用户反馈）

**问题**：用户报告切主题后"主内容卡片的风格没应用进去"。

**根因**：5 个页面（Skills/Tools/Settings/Marketplace/LlmModel）共有 **90+ 处内联 `borderRadius: "8px"/"12px"/"6px"/"999px"`**（grep 实测：Skills 17、Tools 22、Settings 30、Marketplace 21、LlmModel 0）。v3 的 `_enhance.css` 只匹配 `borderRadius: 16/14/4`，**没有覆盖 8/12/6/999 这 4 个实际值**。内联 style 的 specificity 是 0,1,0,0，必须用属性选择器 + `!important` 才能打败。

**修复**（_enhance.css 加 130+ 行）：
1. **Loading Skeleton 段补全 8/12/999**
2. **5 主题 · 页面内联卡片视觉强化段**：
   - 默认：8/12px 中等圆角 + 衬线 display 接管 h1/h2
   - 简约：8px→20px、12px→24px、**所有 button → 9999px pill**
   - 科幻：所有 8/12/6px → 4px + **所有 button → 2px + 全大写 + 等宽字体**
   - 轻拟：8px→24px、12px→28px、**所有 button → 9999px pill + 双向阴影**
   - 漫画：所有 8/12px → 6px、6px → 4px、**button → 4px + Bangers + 3px 粗墨边 + 5px 硬偏移**
3. **default 主题 --radius-button 从 0.375rem → 0.5rem**（与页面 8px 兼容）

**TypeScript 验证**：`npx tsc --noEmit` exit 0
